Windows->Server(nginx+hexo)?

发表于 2016-05-08   |     |  
  • updated 16/8/3：已迁移至github pages。另：在阿里云/万网注册备案的域名，必须解析到阿里云的主机，CNAME到github pages是会被禁止解析,吊销备案的,故把域名转出到namesilo，希望能顺利转出。

当我购买了阿里云学生优惠计划的ECS服务器之后,便开始有了搭建属于自己的个人博客的想法。因为并没有学习过网页前端设计，也没有很重的需求需要LAMP+WordPress，二来学生版的ECS也就1核1G的配置，所以目标就定在了“轻”、“快”上。有了以前在树莓派上搭建LAMP环境的经验，这次在ECS上搭建博客就相比轻松了很多。

为什么要用静态框架呢？

对于访问量、功能需求、界面复杂程度都不大的个人博客而言，静态博客框架具有轻量、快速的优点，首先LAMP中的MySQL、PHP不必要安装，二则Apache可用更轻量的Nginx代替，这样ECS服务器上仅仅需要安装运行Nginx服务器即可！

静态框架搭建的博客常常被部署在github pages上，部署在服务器上的例子比较少，但道理是相通的。静态博客框架常见的有jekyll和hexo，在网上看到很多二者对比的帖子和从jekyll向hexo迁移的帖子，最终决定使用插件支持丰富、生成静态页面快速、支持Markdown的hexo（hexo的第三方主题丰富，其中不少简约派很合我胃口）。

为什么不在本地安装Hexo环境？

原因很简单，其实是一开始我没搞清楚这个博客框架的工作逻辑，以为要安在服务器上，其实服务器需要的仅仅是hexo generat生成的/public文件夹的内容，只需要hexo deploy部署到服务器上即可。既已行差踏错，只好将错就错。其实这样做的好处也有，我可以跨平台来撰写我的博客，不管是PC、安卓还是IOS，只需一个ssh终端即可，不需要重复地配置环境。

在这里遇到了一个比较重要的问题，几经搜索之后尚未解决：Nginx服务器的默认目录修改不生效，在nginx/sites-enabled下的配置文件修改了root目录后，reload后并没有生效。网上搜索问题找到的答案都不能解决问题，推测问题可能是权限，弄得我头昏脑胀。但如果每次hexo g后都要用cp命令来复制public文件夹又非常麻烦，最终受到sites-avaliable和sites-enabled之间关于ln -s软链接的启发，有了如下的非常不可取的做法：

Hexo themes

Hexo官方默认的主题是Landscape，简洁明了，样式比较简陋，可定制程度也不高。于是搜索到了知乎上的一个问题：有哪些好看的Hexo主题？，2015-04-30的一个回答列出了一个榜单，于是我就直接点开了第一名，发现很合胃口，于是就选择了这款国人IIssNan开发的NexT主题：hexo-theme-next。

这款主题的配置文件有很多，能对很多样式进行个人定制，十分好评。

有几个地方可以注意一下：

  • 修改.yml文件时，冒号:后面一定要加一个空格，否则报错。

  • 修改字体配置后若hexo sever预览生效而hexo g后未生效，可以先hexo clean一下再生成。本博客用的字体是参考NexT开发者博客的字体：global: Monda, logo: Lobster Two, codes: PT Mono .

  • 诚如你所见，文章没有段首缩进，可能NexT开发者没有这个习惯，但我个人看着实在难受，网上搜索得到的方法是修改CSS/*styl文件，加上一句.entry{text-indent:2em;}，但NexT的styl文件有很多很多很多，我并没有找到某个特定的文件，加上对CSS不了解，尝试在几个.styl文件下增添了该语句也未见生效。

Markdown

Hexo是支持Markdown语法的，Markdown的使用非常方便，本文就是在PuTTY上vim *.md进行编写的，当然不能实时预览，出错的话要重新vim一下也不是很方便。

有一个解决办法是在本地用支持Markdown语法的编辑器里编写，再copy到服务器上的.md文件里。这里推荐Windows平台的Typora，简洁明了，支持实时预览。这款编辑器同样适配MAC。
